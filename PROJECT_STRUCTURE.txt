═══════════════════════════════════════════════════════════════════════════
FILE DESCRIPTIONS
═══════════════════════════════════════════════════════════════════════════

server.js
  - Main application entry point
  - Express server configuration
  - Middleware setup (security, logging, rate limiting)
  - Route mounting
  - Graceful shutdown handling

src/db.js
  - PostgreSQL connection pool
  - Transaction management
  - Query execution helpers

src/middleware/
  - auth.js: JWT token generation/verification, RBAC authorization
  - errorHandler.js: Global error handling, validation errors

src/models/
  - Implements data access layer
  - All database queries
  - User, Request, Policy, Organization models
  - Audit logging

src/routes/
  - RESTful API endpoints
  - Request validation
  - Response formatting
  - Authorization checks

src/services/
  - riskEngine.js: ML-based scoring, auto-approval logic
  - integrations.js: GitHub, AWS, Okta, Datadog connectors
  - notifications.js: Email templates & sending

src/utils/
  - validation.js: Joi schemas for input validation
  - response.js: Standard response formatting

scripts/
  - migrate.js: Creates all database tables and indexes
  - seed.js: Loads demo data for testing
  - deploy.sh: Production deployment automation

Docker files
  - Dockerfile: Container image definition
  - docker-compose.yml: Local development stack (app + postgres)

Configuration files
  - .env.example: Environment variable template
  - .env.local: Local development settings
  - nginx.conf: NGINX reverse proxy (production)
  - orbit-resolve.service: Systemd service (Linux)
  - k8s-deployment.yaml: Kubernetes deployment manifests
  - .github-workflows-ci.yml: GitHub Actions CI/CD pipeline


═══════════════════════════════════════════════════════════════════════════
DATA FLOW
═══════════════════════════════════════════════════════════════════════════

1. CLIENT REQUEST
   Browser sends HTTP request → CORS check → Rate limiter

2. AUTHENTICATION
   Extract Bearer token → Verify JWT → Load user context

3. ROUTING
   Match URL path → Route handler → Validate input

4. BUSINESS LOGIC
   Query database via models → Execute business rules → Calculate scores

5. DATABASE
   Execute queries → Transaction management → Return data

6. RESPONSE
   Format response → Apply security headers → Send to client

SCALABILITY CONSIDERATIONS
═══════════════════════════════════════════════════════════════════════════

✓ Horizontal scaling: Stateless API servers behind load balancer
✓ Database: Connection pooling, read replicas, sharding ready
✓ Caching: Redis-ready for session & policy cache
✓ CDN: Static assets served via CloudFront/Cloudflare
✓ Message queue: Bull/RabbitMQ for async operations
✓ Monitoring: Structured logging, metrics collection ready
✓ Load balancing: NGINX or cloud provider (ALB, NLB)
