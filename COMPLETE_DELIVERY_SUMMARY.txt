💻 INSTALLATION & SETUP
═══════════════════════════════════════════════════════════════════════════════

PREREQUISITES
  • Node.js 18+ (https://nodejs.org)
  • PostgreSQL 14+ (https://www.postgresql.org)
  • npm or yarn

STEP 1: INSTALL DEPENDENCIES
  $ npm install

STEP 2: CONFIGURE ENVIRONMENT
  $ cp .env.example .env
  Edit .env with your settings:
    - DATABASE_URL (PostgreSQL connection)
    - JWT_SECRET (generate random 32+ chars)
    - STRIPE_SECRET_KEY (if using billing)
    - SMTP credentials (if using email)

STEP 3: CREATE DATABASE
  $ npm run migrate    # Creates all tables & indexes
  $ npm run seed       # Loads demo data

STEP 4: START SERVER
  Development:
    $ npm run dev      # Auto-reload on changes
  
  Production:
    $ npm start        # Normal startup

STEP 5: VERIFY IT WORKS
  $ curl http://localhost:3000/health
  Expected: { "status": "healthy", "uptime": ... }

STEP 6: TEST LOGIN
  Email: admin@acme.com
  Password: password123
  URL: http://localhost:3000/login.html

📊 DATABASE SCHEMA
═══════════════════════════════════════════════════════════════════════════════

organizations
  - id (UUID primary key)
  - name, owner_id
  - subscription_status, subscription_tier
  - created_at, updated_at

users
  - id (UUID), org_id (foreign key)
  - email (unique), password_hash
  - name, role (employee|manager|admin|exec)
  - created_at, updated_at

requests
  - id, org_id, user_id, policy_id
  - title, description, type
  - requestor_role, target_resource, urgency
  - status (pending|approved|denied|provisioning|completed)
  - confidence (0-100), approver_id, approval_notes
  - created_at, updated_at

exceptions
  - id, org_id, request_id
  - reason, policy_conflict (JSON), recommendation (JSON)
  - status, resolved_by, resolution_notes
  - created_at, updated_at

policies
  - id, org_id
  - name, description
  - threshold, auto_approval_rate
  - created_at, updated_at

audit_logs
  - id, org_id, user_id
  - action, entity_type, entity_id
  - details (JSON), ip_address
  - created_at

🆘 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Issue: "Cannot connect to database"
  Solution: Check DATABASE_URL in .env, ensure PostgreSQL is running

Issue: "Port 3000 already in use"
  Solution: Kill process: lsof -i :3000 | kill -9 <PID>

Issue: "JWT token invalid"
  Solution: Regenerate JWT_SECRET in .env, restart server

Issue: "Email not sending"
  Solution: Check SMTP credentials in .env, verify SendGrid/mail service

See DEPLOYMENT.md for more troubleshooting tips.
